You need to fork a process for every command
bc in order to execute the command, You
need to use the execv function.
Which if not forked before calling it,
the parent process will be replaced by 
the process you are trying to execute.
By forking, we keep the parent going
after execv is called in the child.


syntax:
< infile com args | com2 args2 > outfile

'<' here redirects the contents of infile to the stdin
and '>' redirects the output of com2 args2 to outfile.
'|' redirects the output of com args to the input of com2 args2.

'>>' is like '>' but it appends the result to the end of outfile.
'<<' reads stdin or infile until some limiter is reached.

e. g.
cat << LIM >> outfile
> This is line 1
> This is line 2
> LIM

cat << LIM >> outfile
> This is the last line being appended
> LIM

so outfile should be:

This is line 1
This is line 2
This is the last line being appended


Functions:

access(char *pathname, int mode) 
- checks whether file can be accessed in 'mode' mode.

dup2(int oldfd, int newfd)
- replaces oldfd with newfd so that newfd and oldfd refer to the same file
that oldfd refers to.
- e. g. say infile.txt has fd 3.  Then after dup2(3, 0), the file descriptor
for stdin now refers to infile.txt.

pipe(int pipefd[2])
- creates a communication channel where pipefd[0] is the reading end and pipefd[1]
is the writing end of the channel.
- whatever you write to pipefd[1] is accessible from pipefd[0].

execve(char *path, char *argv, char **envp)
- executes the program found in path, using arguments argv
where argv[0] is the command/path-to-file.
- envp is the list of envvars as strings.

unlink(char *pathname)
- removes a closed file
